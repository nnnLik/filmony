# План: новый флоу создания карточки

## Что меняем
- Упрощаем Ленту: убираем устаревший текст/блоки «до появления карточек», оставляем чистый entry-point в создание карточки.
- Переводим экран создания карточки в нумерованный пошаговый wizard (1→N) вместо длинной формы на одном экране.
- Используем уже существующий backend `POST /api/films/resolve` как шаг получения фильма по URL (он внутри вызывает Kinopoisk API и парсит id из ссылки).
- Сохраняем финал: после `Готово` создаём карточку через `POST /api/cards` и переводим пользователя в профиль.

## Поведение и валидация по шагам
- Этап 1 (URL):
  - Пустой ввод: показать понятную ошибку «Вставьте ссылку на Кинопоиск».
  - Неразбираемая ссылка/неверный домен/нет id: показываем текст ошибки от backend в дружелюбной форме, остаёмся на том же этапе.
- Этап 2 (подтверждение фильма):
  - Успех: рендерим постер + название (опционально год), спрашиваем подтверждение.
  - «Не тот фильм»: сбрасываем выбранный фильм и возвращаем на этап ввода URL без выхода со страницы.
- Этап 3 (оценка и контекст):
  - Центральный рейтинг 1–10, шаг 0.5 (как сейчас).
  - Ниже визуально оформить категории тегов (в т.ч. «С кем смотрели») как компактные цветные прямоугольные chips.
- Этап 4 (свои теги):
  - До 5 тегов, дедупликация, удаление по клику.
- Этап 5 (мок):
  - Инфо-блок про будущую фичу шаринга + `Готово`.

## Тесты и проверка
- Frontend:
  - Проверить ручной сценарий: URL -> resolve -> подтверждение -> рейтинг/теги -> мок -> создание -> переход в профиль.
  - Негативные кейсы: пустой URL, не-kinopoisk URL, URL без id, ошибка сети.
- Backend:
  - Переиспользуется существующая логика parse/resolve; при необходимости только уточнить пользовательские тексты на фронте без изменения API.
