# Feature: вкладка «Поиск» (локальный каталог + люди + рекомендации)

## Цель

- Отдельная вкладка в Mini App для **локального** поиска тайтлов (`film`) и пользователей (`user`) по строке запроса (вариант A — только данные Filmony).
- Блок **рекомендаций людей** до ввода запроса: три источника с лимитами и дедупликацией.
- Пустое состояние по тайтлам: *«Пока никто не добавлял этот тайтл. Можете первым — через ссылку с Кинопоиска.»* + CTA «Добавить карточку».

## User story

Как залогиненный пользователь, хочу открыть «Поиск», чтобы найти тайтл в каталоге или человека по имени/slug, и видеть подсказки «кого посмотреть» до ввода запроса.

## API

### `GET /api/search`

- **Auth:** обязательна (`CurrentUser`).
- **Query:** `q` — обязательно, trim, длина **2–64** символов; `limit_films`, `limit_users` — опционально, целые **1–30**, дефолт **15** каждый, сервер обрезает сверх максимума.
- **Ответ:** `{ "films": [...], "users": [...] }` — компактные DTO для списка.
- **Поиск тайтлов:** `film.title ILIKE '%' || q || '%'` (регистронезависимо через `ilike`).
- **Поиск людей:** `display_name`, `username`, `profile_slug` — `ILIKE` по OR; **не** подбирать пользователей по `telegram_user_id` в строке.

### `GET /api/search/suggestions`

- **Auth:** обязательна.
- **Ответ:** три массива + опционально метка источника для UI:
  - `mutual_circle` — пользователи с наибольшим пересечением «я подписан на X» и «кандидат подписан на X» (`user_subscription`), исключая себя и уже подписанных; лимит **5**.
  - `popular_authors` — топ авторов `movie_card` за **7 дней** по полю **`created_at`** (кол-во новых карточек); лимит **5**; исключить себя.
  - `random_with_cards` — случайные пользователи с ≥1 карточкой; лимит **5**; исключить себя.
- **Дедуп:** порядок приоритета `mutual_circle` → `popular_authors` → `random_with_cards`; итог до **15** уникальных `user_id` в ответе (каждый пользователь в одном месте — в первом непустом источнике по приоритету, либо три списка как пришли с дедуп на фронте — **контракт:** три ключа в JSON, дедуп на бэкенде внутри каждого списка нет дублей; пересечение между списками допускается **только если** фронт дедупит — проще **дедуп на бэкенде**: каждый пользователь максимум в одном массиве (первый подходящий источник). Уточнение: отдаём **три массива без пересечения** (после дедупа по приоритету mutual заполняет, popular добирает без уже выбранных, random — без уже выбранных).

## Acceptance criteria

- [ ] Без сессии оба эндпоинта → **401**.
- [ ] `q` длина 0/1 → **422** с понятным `detail`.
- [ ] Поиск возвращает только существующие строки; пустые массивы допустимы.
- [ ] Suggestions исключают текущего пользователя; не включают уже подписанных в `mutual_circle` логике (кандидаты — потенциальные новые подписки).
- [ ] `pytest` покрывает поиск и suggestions (в Docker).
- [ ] Фронт: вкладка «Поиск», debounce, пустые состояния, переходы на `/films/:id` и `/u/:id`.
