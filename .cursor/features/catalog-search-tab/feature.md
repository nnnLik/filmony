# Feature: вкладка «Поиск» (локальный каталог + люди + рекомендации)

## Цель

- Отдельная вкладка в Mini App для **локального** поиска **карточек** пользователей и **профилей** по строке запроса (данные только Filmony).
- Совместимость: если API ещё отдаёт только **`films`**, клиент продолжает показывать список тайтлов с переходом на `/films/:id`.
- Блок **рекомендаций людей** до ввода запроса: три источника с лимитами и дедупликацией (без изменений).
- Пустое состояние по основной выдаче: подсказка добавить первую карточку + CTA «Добавить карточку».

## User story

Как залогиненный пользователь, хочу открыть «Поиск», чтобы найти чужую карточку по тексту или человека по имени/slug, и видеть подсказки «кого посмотреть» до ввода запроса.

## API

### `GET /api/search`

- **Auth:** обязательна (`CurrentUser`).
- **Query:** `q` — обязательно, trim, длина **2–64** символов; при карточном поиске — `limit_cards`, `limit_users`; при legacy — `limit_films`, `limit_users` — опционально, целые **1–30**, сервер задаёт дефолты и обрезает сверх максимума.
- **Ответ:** `{ "cards": [...], "films": [...], "users": [...] }` — `cards` основной список; `films` может отсутствовать или быть пустым после миграции.
- **Поиск карточек:** по текстовым полям карточки (правила совпадают с бэкендом после миграции).
- **Поиск людей:** `display_name`, `username`, `profile_slug` — `ILIKE` по OR; **не** подбирать пользователей по `telegram_user_id` в строке.

### `GET /api/search/suggestions`

- **Auth:** обязательна.
- **Ответ:** три массива и дедуп между секциями по приоритету `mutual_circle` → `popular_authors` → `random_with_cards` (каждый пользователь не более чем в одном списке).

## Acceptance criteria

- [ ] Без сессии оба эндпоинта → **401**.
- [ ] `q` длина 0/1 → **422** с понятным `detail`.
- [ ] Поиск возвращает только существующие строки; пустые массивы допустимы.
- [ ] Suggestions исключают текущего пользователя; не включают уже подписанных в `mutual_circle` логике (кандидаты — потенциальные новые подписки).
- [ ] `pytest` покрывает поиск и suggestions (в Docker).
- [ ] Фронт: вкладка «Поиск», debounce, пустые состояния, переходы на **`/cards/:id`** для основной выдачи (и `/films/:id` при legacy `films` без `cards`), `/u/:id` для людей.

## Out of scope

- Внешний поиск Кинопоиска в этой вкладке (отдельный поток `/api/catalog/search` при создании карточки).
- pg_trgm в первой итерации (опционально позже).
