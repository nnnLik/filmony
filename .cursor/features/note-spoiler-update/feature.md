# Лимит заметок и спойлеры

## Scope
- Увеличить лимит `watch_note` карточки с 500 до 1000 символов (backend + frontend + миграция БД).
- Добавить inline-спойлеры в заметках карточек, комментариях и постах ленты.
- Кнопка «спойлер» в composer: оборачивает выделение в `⟦S⟧…⟦/S⟧`, снимает обёртку при повторном нажатии, без выделения вставляет шаблон.

## Acceptance criteria
- API принимает `watch_note` до 1000 символов; валидация одинакова в schemas и сервисах.
- Спойлеры валидируются на backend (баланс маркеров, без вложенности, не пустые).
- Рендер скрывает текст до клика; работает в `CommentBodyWithReactionTokens` для заметок, комментариев, постов.
- Кнопка спойлера доступна во всех composer-экранах (заметка, комментарии, пост).
- Тесты: `test_spoiler_tokens.py`, API cards routes, `spoilerTokens.test.ts`.
- `npm run lint`, `npm run build` проходят.

## Out of scope
- Rich-text / markdown редактор.
- Вложенные спойлеры.
- Увеличение лимита `share_comment` (остаётся 500).
