# Feature Request

## Metadata
- Feature slug: `movie-card-edit-delete`
- Author: Codex
- Created at: 2026-05-06 20:40 UTC
- Priority: high
- Target area: fullstack

## Problem
- В карточке фильма нет меню действий для владельца.
- Пользователь не может исправить оценку и теги после создания карточки.
- Пользователь не может удалить карточку через UI.

## Scope
- In scope:
  - Owner-only backend API для редактирования и удаления карточки фильма.
  - UI-меню действий на detail-экране карточки (`Редактировать`, `Удалить`).
  - Экран редактирования `rating`, `company`, `mood_before`, `mood_after`, `custom_tags`.
  - Confirm-удаление карточки и обновление клиентского кэша профиля.
- Out of scope:
  - Редактирование привязанного фильма (`film_id`, `kinopoisk_id`, `genres`).
  - Изменение лайков/комментариев, кроме каскадного удаления вместе с карточкой.

## Functional Requirements
- [ ] `PATCH /api/cards/{card_id}` обновляет `rating` и теги карточки владельца.
- [ ] `DELETE /api/cards/{card_id}` удаляет карточку владельца.
- [ ] Для чужой карточки `PATCH`/`DELETE` возвращают `403`.
- [ ] На detail-экране владелец видит меню `...` с пунктами `Редактировать` и `Удалить`.
- [ ] Удаление требует пользовательского confirm.

## Acceptance Criteria
- [ ] Владелец может отредактировать оценку и все теги карточки.
- [ ] Владелец может удалить карточку и вернуться в профиль без stale данных.
- [ ] Чужой пользователь не может редактировать/удалять карточку (policy `403`).
- [ ] API и UI не ломают текущие сценарии просмотра карточек и комментариев.

## Constraints
- Technical constraints:
  - Сервисная логика в backend должна быть вынесена в отдельные service-классы.
  - Проверки backend выполнять в Docker-окружении проекта.
- Product/design constraints:
  - UX меню действий соответствует Telegram Mini App паттерну (как в приложенном скриншоте).
  - Редактирование ограничено рейтингом и тегами.
