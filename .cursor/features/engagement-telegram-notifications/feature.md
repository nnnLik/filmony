# engagement-telegram-notifications

## Purpose

Уведомления в Telegram (личка бота) при:

1. **Ответе на комментарий** к карточке — автор родительского комментария получает сообщение (если ответ не от себя).
2. **Новой реакции** на карточку или на комментарий — владелец карточки / автор комментария получает сообщение (только при **добавлении** реакции, не при снятии; при реакции «на себя» уведомление не шлём).

Сообщение: кто сделал действие, какое событие, при реакции — метка реакции из каталога; опционально **стикер** из env перед текстом.

## Acceptance criteria

- После `POST .../comments` с `parent_comment_id` получатель — автор родительского комментария — получает DM (если не сам автор ответа).
- После `POST /api/reactions` при **первичной** установке реакции владелец цели получает DM; при снятии той же реакции — **нет** DM.
- Ошибки Telegram (нет Start, сеть) **не ломают** HTTP-ответ API; только логирование.
- Pytest: мок Bot API; сценарии skip self, reply, reaction add/remove.

## Out of scope

- Celery / очередь.
- Отдельный webhook для входящих сообщений бота.
- Уведомление автору карточки о **первом** корневом комментарии без parent (не запрошено).
