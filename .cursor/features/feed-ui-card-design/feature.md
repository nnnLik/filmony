# Feature Request

## Metadata
- Feature slug: `feed-ui-card-design`
- Author: Codex
- Created at: 2026-05-06 20:52 UTC
- Priority: high
- Target area: frontend

## Problem
- В ленте пока не зафиксирован единый UX карточки, из-за чего реализация может расползтись по стилям и сценариям.
- Нужно заранее определить, какие данные видны в карточке и какие действия доступны без перехода в detail.
- Текущая задача требует компактного отображения тегов и комментариев, чтобы не перегружать ленту.

## Scope
- In scope:
  - Визуальная структура карточки в ленте: постер, название, оценка, теги, preview комментариев.
  - UX для просмотра всех комментариев через переход в страницу карточки.
  - UX для быстрого комментария из ленты (expand input -> send).
  - Правила отображения системных и пользовательских тегов.
  - Состояния loading/empty/error для ленты и карточек-скелетонов.
- Out of scope:
  - Финальный алгоритм ранжирования и смешивания источников ленты.
  - Backend-изменения по рекомендательной выдаче.

## Concrete Requirements
- [ ] Карточка ленты обязательно показывает: постер, название фильма, оценку автора карточки.
- [ ] Карточка показывает системные теги в нетекстовом формате (иконки/чипы/визуальные маркеры), а не простым списком текста.
- [ ] Пользовательские кастомные теги показываются ограниченно: не более 2-4 тегов + индикатор остатка (`+N`).
- [ ] Карточка показывает 1-2 последних комментария как preview.
- [ ] Есть CTA `Все комментарии`, ведущий в detail-страницу карточки.
- [ ] Есть CTA `Комментировать` в карточке ленты: при нажатии раскрывается поле ввода, отправка комментария происходит без ухода со страницы.
- [ ] После отправки комментария карточка локально обновляет preview и счетчик комментариев без полного рефреша ленты.
- [ ] Компоненты должны базироваться на `@telegram-apps/telegram-ui` и текущих паттернах проекта.

## Success Criteria
- [ ] Пользователь может понять содержимое карточки (что за фильм и чья оценка) за 1-2 секунды сканирования ленты.
- [ ] Системные теги визуально различимы и читаемы без текстовой перегрузки.
- [ ] Кастомные теги не ломают layout при большом количестве.
- [ ] Быстрый комментарий из ленты отправляется стабильно и не сбрасывает позицию скролла.
- [ ] Переход к полным комментариям работает из каждой карточки.
- [ ] На мобильном экране Telegram Mini App карточка не имеет горизонтального скролла.

## MVP (Minimal First Version)
- [ ] Новый/обновленный компонент карточки ленты с обязательными полями: постер, название, оценка.
- [ ] Системные теги отображаются как цветные иконки/чипы (минимум 1 визуальный стиль, согласованный с ui-kit).
- [ ] Кастомные теги: до 2 штук + `+N`.
- [ ] Комментарии: показываем до 2 последних, без сложной сортировки.
- [ ] Inline-комментирование: кнопка, раскрытие поля, отправка, оптимистичное добавление в preview.
- [ ] Базовые состояния: loading skeleton карточки и fallback при ошибке отправки комментария.

## Constraints
- Technical constraints:
  - Переиспользовать существующие frontend-паттерны и API-слой комментариев карточки.
  - Не добавлять тяжелые рендер-паттерны, которые ухудшат производительность бесконечной ленты.
- Product/design constraints:
  - Интерфейс должен оставаться компактным и пригодным для быстрого скролла.
  - Системные теги не отображаются как plain-text список.
