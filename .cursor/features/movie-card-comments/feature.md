# Feature: movie-card-comments

## Metadata
- Feature slug: `movie-card-comments`
- Source: chat request (2026-05-06)
- Priority: P1
- Target area: backend

## Problem
Нужно дать возможность любому пользователю читать комментарии под карточкой фильма, а авторизованным пользователям — оставлять комментарии и отвечать на комментарии любой глубины.

## Scope
- In scope: backend модель/сервисы/API/тесты для комментариев под `movie_card`.
- Out of scope: frontend UI для комментариев, лайки, уведомления.

## Functional Requirements
- [ ] Публичный `GET` корневых комментариев по карточке с курсорной пагинацией.
- [ ] Публичный `GET` ответов конкретного комментария с курсорной пагинацией.
- [ ] `POST` комментария/ответа только для авторизованных пользователей.
- [ ] Валидация текста комментария (`trim`, не пустой, max length).
- [ ] Проверка, что `parent_comment_id` относится к той же карточке.

## Acceptance Criteria
- [ ] Без авторизации можно читать комментарии и ответы (`200`).
- [ ] Без авторизации нельзя создавать комментарии (`401`).
- [ ] Ответы поддерживают неограниченную вложенность.
- [ ] Ошибки валидации/связности parent возвращают корректные `4xx`.
