# Profile statistics redesign

## Цель

Переработать вкладку «Статистика» в профиле: три внутренних подвкладки (**Обзор**, **Вкусы**, **Рейтинги**), понятное распределение существующих метрик без потери данных, мобильный UX без горизонтального скролла.

## В scope (этот slug)

1. Иерархия подвкладок и явное назначение каждого раздела статистики.
2. Карта метрик по подвккладкам (существующие API и блоки данных сохранены).
3. Дальнейшие итерации (визуальная полировка, новые фильтры, deep-link на подвкладку) — опционально, отдельными задачами.

## Acceptance criteria

- На экране статистики есть сегментированный переключатель на три пункта: **Обзор**, **Вкусы**, **Рейтинги** (или эквивалент в стиле проекта).
- **Обзор**: KPI (всего карточек, средняя оценка), короткий текстовый резюме, главный высокосигнальный график — распределение оценок.
- **Вкусы**: распределение оценок, полярность вкуса, теги, «с кем смотрю», «настроение после» — все блоки сохранены.
- **Рейтинги**: топ по оценке, самые низкие оценки (списки-ссылки на карточки), распределение по годам («По годам темы») — сохранены.
- Поведение загрузки и источники данных те же (`getUserMovieCardStats`), без изменения контракта по этой задаче.
- Интерфейс мобильный, без принудительной горизонтальной прокрутки для графиков столбиков там, где это контролируется компонентом.
