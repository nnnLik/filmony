# Feature: список «К просмотру» (`watchlist`)

Публичный список фильмов пользователя без оценённой карточки: фильм уже есть в каталоге (после резолва с Кинопоиска или из других экранов), оценку и контекст пользователь заполняет позже.

## Цель

- Отложить просмотр без создания `MovieCard`: не заполнять рейтинг и анкету на этом шаге.
- Показать список на профиле (свой и чужой) отдельно от оценённых карточек.
- На странице фильма дать те же действия: оценить или добавить/убрать из списка.

## Реализованный объём

### Backend

- Таблица связей пользователь–фильм (уникальная пара `user_id` + `film_id`).
- Сервисы: добавление, удаление, постраничный список для профиля, проверка «в списке ли» для текущего пользователя.
- HTTP (типовые пути проекта):
  - список на профиле пользователя;
  - мутации для текущего пользователя (`POST` добавление по `film_id`, `DELETE` удаление);
  - счётчик в данных профиля (`watchlist_count`).
- Инвариант: при успешном создании оценённой карточки для той же пары фильм записывается из списка снимается в одной транзакции с созданием карточки.

Подробности контрактов и файлов см. в тестах `backend/src/tests/api/test_watchlist_routes.py` и связанных сервисах в `backend/src/services/watchlist/`.

### Frontend

| Место | Поведение |
|------|-----------|
| `CreateCardPage` шаг 2 «Подтверждение фильма» | Три кнопки без длинных подсказок: **Оценить просмотр** → шаг 3; **К просмотру** → `POST` watchlist и переход в профиль на «К просмотру»; **Другой фильм** → шаг 1. При «ремиксе» с чужой карточки — одна короткая строка про отдельную карточку. |
| `ProfilePage` / `PublicProfilePage` | Внутри «Фильмы»: сегменты «Оценённые» и «К просмотру», сетка постеров для watchlist. |
| `FilmDetailPage` (`/films/:filmId`) | **Оценить просмотр** → `/cards/new?filmId=…`; при отсутствии в списке — **К просмотру**; если в списке — **Убрать из списка**. Ошибки мутаций — под кнопками. |

## Ограничения и краевые случаи

- Фильм уже в списке — повторное добавление даёт конфликт (на UI показывается понятное сообщение).
- У пользователя уже есть оценённая карточка по этому фильму — добавление в watchlist не допускается (сообщение об ошибке с бэкенда).
- Лента приложения по-прежнему строится только по оценённым карточкам, не по watchlist.
