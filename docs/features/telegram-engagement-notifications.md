# Telegram engagement notifications

## Summary
- Пользователь может **отправить свою карточку фильма** пользователю из списка **своих подписок**; адресат получает **короткое сообщение в боте**: что произошло, от кого, ссылка в Mini App.
- В перспективе: уведомления при **реакции** на карточку (автору карточки) и на **комментарий** (автору комментария). В backend пока нет сущности «реакция» — события закладываются в спеке.

## User flows
1. **Share карточки**  
   Автор после создания (или с экрана карточки) выбирает получателя только среди **following** → API валидирует → в очередь ставится отправка `sendMessage` получателю.

2. **Реакция на карточку** (v2)  
   Пользователь ставит эмоцию на чужую карточку → владелец карточки получает уведомление в боте (если не self-react).

3. **Реакция на комментарий** (v2)  
   Пользователь ставит эмоцию на комментарий → автор комментария получает уведомление (если не self-react).

## Message format (Telegram)
- Минимум текста: **действие** + **имя инициатора** + **одна ссылка** на Mini App (карточка / при необходимости комментарий).
- Без вложений и сложной разметки в v1.

## Правила и защита
- Share только на пользователя из **моих подписок**; не себе.
- **Dedupe** по ключу события (например та же карточка → тот же получатель) с TTL.
- **Rate limits** на число исходящих уведомлений с одного аккаунта за интервал.
- Ошибки доставки (бот заблокирован, нет чата) **не ломают** основной сценарий; статус фиксируется для диагностики.

## Важные ограничения
- Получатель должен иметь **начатый диалог с ботом** (`/start`), иначе личное сообщение может быть недоступно.
- Исходящая доставка должна быть **асинхронной** (worker), не в синхронном ответе HTTP.
- См. также черновик: `.cursor/features/009-telegram-notifications.md`.

## MVP
- Реализовать только **share карточки** + инфраструктура отправки и тесты.
- Реакции — после появления модели и API реакций.

## API (предлагаемый контракт)
- `POST /api/cards/{card_id}/share`  
  Body: `{ "recipient_user_id": "<uuid или int>" }`  
  Ответы: `200` (принято к доставке), `403` (не подписка), `401`, `404`.
