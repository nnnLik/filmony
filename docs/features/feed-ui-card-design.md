# Feed UI Card Design

## Goal
- Определить и зафиксировать UX/UI ленты и карточки так, чтобы реализацию можно было начать немедленно.

## Concrete Requirements
- Карточка обязательно содержит постер, название и оценку автора.
- Системные теги показываются в визуальном формате (иконки/чипы), а не plain text.
- Кастомные теги отображаются ограниченно: до 2-4 штук + `+N`.
- В карточке показываются 1-2 комментария preview.
- Для полного списка комментариев используется переход в detail карточки.
- Комментарий можно отправить прямо из ленты через раскрывающийся composer.

## Success Criteria
- Карточка быстро сканируется пользователем и не перегружена текстом.
- Системные и пользовательские теги визуально различаются и читаемы.
- Inline-комментарий отправляется без потери позиции скролла.
- Layout стабилен на мобильном экране Telegram Mini App.

## MVP
- Компонент карточки с: постер, title, rating.
- Визуальные системные теги.
- До 2 кастомных тегов + `+N`.
- До 2 preview комментариев.
- Inline поле комментария с отправкой и локальным обновлением preview.
- Loading skeleton и базовый error-state отправки комментария.
